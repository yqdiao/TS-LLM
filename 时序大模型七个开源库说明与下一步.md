# 七个时序大模型开源库说明与下一步行动

本文档基于你提供的四个 GitHub 链接，简要概括各库的定位与用法，并对照你的「分钟级股票小模型 vs 时间序列大模型」对比方案，给出**下一步该做什么**。

---

## 一、七个时序模型仓库概览

### 1. [TimesFM](https://github.com/google-research/timesfm)（Google Research）

| 项目 | 说明 |
|------|------|
| **定位** | 预训练的时间序列**基础模型**（Foundation Model），decoder-only，面向通用时序预测。 |
| **论文** | *A decoder-only foundation model for time-series forecasting* (ICML 2024)。 |
| **最新版本** | TimesFM 2.5：约 200M 参数，最大 context 16k，支持 horizon 最长 256（可选 quantile 头到 1k）。 |
| **输入输出** | 输入：一条或多条时间序列（如 `list of 1D arrays`）；输出：点预测 + 可选分位数预测。 |
| **安装与使用** | `pip install` 或 clone 后 `uv pip install -e .[torch]`；从 Hugging Face 加载 `google/timesfm-2.5-200m-pytorch`，配置 `ForecastConfig` 后调用 `model.forecast(horizon=..., inputs=[...])`。 |
| **与分钟级股票的契合度** | **高**。支持多序列、长 context、多步 horizon；可直接把「过去 N 分钟」的 5 元特征（或 mid_price/return）做成多变量序列，预测未来 30 分钟（如 horizon=30 或先降采样再预测）。无需自己训练，零样本/少样本即可对比。 |

**参考**：[GitHub - google-research/timesfm](https://github.com/google-research/timesfm)，[Hugging Face - TimesFM](https://huggingface.co/collections/google/timesfm-release-66e4be5fdb56e960c1e482a6)。

---

### 2. [Amazon Forecast Samples](https://github.com/aws-samples/amazon-forecast-samples/tree/master)（AWS）

| 项目 | 说明 |
|------|------|
| **定位** | **云服务示例**：演示如何用 Amazon Forecast **API** 做预测（导入数据、训练 predictor、生成 forecast）。 |
| **内容** | Workshops、Notebooks、MLOps 流水线示例；数据多为销量、能源等，**不是**本地开源的「大模型代码」。 |
| **状态** | 仓库于 2025 年 5 月被归档（read-only）。 |
| **与分钟级股票的契合度** | **低**。需要 AWS 账号、S3、Forecast 服务；且是「调 API」而非「跑一个可复现的时序大模型代码」，不利于你控制输入格式、固定评估协议和做「小模型 vs 同一大模型」的公平对比。 |

**建议**：做「小模型 vs 时序大模型」对比时，**可暂不采用**；若你已有 AWS 且希望对比「云服务 vs 本地小模型」，再考虑把分钟级数据整理成 Forecast 要求的 target + item 等格式。

### 3. [PatchTST](https://github.com/yuqinie98/PatchTST)（ICLR 2023）

| 项目 | 说明 |
|------|------|
| **定位** | **Transformer 时序预测**：把时间序列切成 patch（子序列）作为 token，channel-independent，适合长序列。 |
| **论文** | *A Time Series is Worth 64 Words: Long-term Forecasting with Transformers*。 |
| **特点** | 有监督 + 自监督两种用法；支持多变量、可调 look-back 与 prediction length；已集成到 GluonTS、NeuralForecast、tsai。 |
| **数据与训练** | 需自备 CSV（如 ETT、Weather）；脚本在 `scripts/PatchTST/`，例如 `weather.sh`；超参可调 patch 长度、look-back、预测长度。 |
| **与分钟级股票的契合度** | **高**。多变量、长序列友好；你可把「过去 30–60 分钟」的 [volume, best_ask, size_ask, best_bid, size_bid]（及 mid_price、return）作为多 channel 输入，预测未来 30 步（分钟）收益或方向。需要**自己训练**，算作「中等规模时序模型」，便于与 LightGBM/小 LSTM 和 TimesFM 一起对比。 |

**参考**：[GitHub - yuqinie98/PatchTST](https://github.com/yuqinie98/PatchTST)。

---

### 4. [Informer2020](https://github.com/zhouhaoyi/Informer2020)（AAAI 2021 Best Paper）

| 项目 | 说明 |
|------|------|
| **定位** | **长序列时序 Transformer**：ProbSparse 注意力，降低长序列计算量。 |
| **论文** | *Informer: Beyond Efficient Transformer for Long Sequence Time-Series Forecasting*。 |
| **特点** | 支持多变量 (M/S/MS)、可配置 seq_len / label_len / pred_len；freq 支持 minutely (`t`)。 |
| **数据与训练** | 数据放 `data/ETT/` 等；`main_informer.py` 指定 `--data`、`--seq_len`、`--pred_len`、`--freq t` 等即可训练与预测。 |
| **与分钟级股票的契合度** | **高**。多变量、支持分钟频；你可把分钟级 5 元特征做成 CSV，seq_len=60、pred_len=30 做「未来 30 分钟」预测。同样需要**自己训练**，适合作为「中等规模时序模型」与 TimesFM（零样本）和小模型对比。 |

**参考**：[GitHub - zhouhaoyi/Informer2020](https://github.com/zhouhaoyi/Informer2020)。

---

### 5. [Chronos](https://github.com/amazon-science/chronos-forecasting)（Amazon Science）

| 项目 | 说明 |
|------|------|
| **定位** | 预训练时序预测模型族：Chronos-2（零样本单变量/多变量/带协变量）、Chronos-Bolt（patch 编码、更快更省显存）。 |
| **模型规模** | 8M--710M 多种规格；Hugging Face 提供 `amazon/chronos-2` 等。 |
| **使用** | `pip install chronos-forecasting`；`Chronos2Pipeline.from_pretrained` + `predict_df`，支持 `prediction_length`、协变量等。 |
| **与分钟级股票的契合度** | **高**。零样本即可；将分钟级 5 元特征或 mid_price 作为 target/context，`prediction_length=30` 得未来 30 步预测。 |
| **在方案中的角色** | 与 TimesFM 并列的「预训练零样本时序模型」，可一并对比。 |

**参考**：[GitHub - amazon-science/chronos-forecasting](https://github.com/amazon-science/chronos-forecasting)，[Hugging Face - Chronos](https://huggingface.co/amazon/chronos-2)。

### 6. [Lag-Llama](https://github.com/time-series-foundation-models/lag-llama)（Time-Series Foundation Models）

| 项目 | 说明 |
|------|------|
| **定位** | 面向**概率时序预测**的开源基础模型（基于 Llama 架构）；支持零样本预测与微调，任意频率、任意预测长度。 |
| **使用** | 官方提供 Colab 零样本与微调示例；建议调节 context length（32/64/128/256/512/1024）与 RoPE scaling 以适配分钟级。 |
| **与分钟级股票的契合度** | **高**。零样本或在本项目 1 月数据上微调后，用历史分钟序列预测未来 30 步分布。 |
| **在方案中的角色** | 预训练 + 可微调的时序基础模型，可与 TimesFM/Chronos 零样本及 PatchTST/Informer 一起参与「零样本 vs 微调 vs 小模型」对比。 |

**参考**：[GitHub - time-series-foundation-models/lag-llama](https://github.com/time-series-foundation-models/lag-llama)。

---

### 7. [DeepLOB](https://github.com/S-razmi/DeepLOB)（订单簿专用，可选）

| 项目 | 说明 |
|------|------|
| **定位** | 基于论文「DeepLOB: Deep Convolutional Neural Networks for Limit Order Books」的实现，面向**订单簿（LOB）**数据；该仓库为 BTC 永续 LOB 的 1 秒频数据上的 CNN/TCN 实现（TensorFlow）。**非大语言模型**，而是领域专用的深度学习基线。 |
| **与分钟级股票的契合度** | **高**。best_ask、best_bid、size 即为 LOB 核心字段；可将分钟级 LOB 特征构造为模型输入，预测未来价格方向或收益区间。 |
| **在方案中的角色** | 可选；代表「订单簿/高频场景下的专用深度学习模型」，用于补充对比或论证在 LOB 场景下轻量模型是否足以媲美专用结构。 |

**参考**：[GitHub - S-razmi/DeepLOB](https://github.com/S-razmi/DeepLOB)。

### 适用性说明：时序模型与股票价格预测

上述模型（含 TimesFM、PatchTST、Informer、Chronos、Lag-Llama、DeepLOB 等）中，通用时序模型均可用于股票价格时间序列预测，但**均非专为股票价格设计**，故属「可用但有前提」而非「完全、无条件适用」。TimesFM 为通用时序基础模型，在股票上属零样本迁移，效果取决于与预训练分布的接近程度。Amazon Forecast 为通用预测云服务，可做股价预测但非股票专用且依赖 AWS。PatchTST、Informer 为通用长序列模型，论文实验多为电力、天气等；用于股价预测需在**本数据集上自行训练或微调**，并注意金融特性（非平稳、波动聚集、交易时段等）。预训练数据多为日/小时频，分钟级高频价量的分布与噪声与常见预训练数据不同，零样本或直接迁移未必最优。若在本实验中**小模型不逊于或优于**这些大模型，可说明在分钟级股票场景下通用时序大模型并非「完全适用」，盲目上大模型可能是资源浪费；若要提高在股票预测上的适用性，建议在自有股票数据上训练或微调（如 PatchTST/Informer），或选用面向金融/高频的时序模型。

## 二、在你方案中的角色划分

结合你已有的「分钟级股票小模型 vs 时间序列大模型」方案，可以这样归类：

- **小模型（证明「小就够用」）**：LightGBM/XGBoost、小 LSTM/TCN、线性/AR。  
- **时序大模型 / 专长模型（被对比对象）**：  
  - **TimesFM / Chronos / Lag-Llama**：预训练基础模型，**零样本/少样本**即可跑（Lag-Llama 另支持微调），代表「现成 TS-Foundation Model」；  
  - **PatchTST / Informer**：需要在你数据上**训练**，代表「可训练的时序 Transformer」，参数量与计算量都大于 LightGBM/小 LSTM；  
  - **DeepLOB**（可选）：订单簿专用 CNN/TCN，与 best_ask/best_bid/size 数据高度相关，可作为领域专用基线。

Amazon Forecast 更偏云服务示例，若不做 AWS 对比可先不纳入。

## 三、下一步该做什么（按顺序）

### 第一步：把分钟级数据与标签准备好（优先）

- 在原始 CSV 里**确认并扩展列**：除现有 `time_str, best_ask, best_bid` 外，确认 **trade volume**、**size of best ask**、**size of best bid** 的列索引（必要时看数据说明或逐列核对一条 CSV）。  
- 在 PySpark 中：  
  - 按 (date, stock_code, minute) 聚合，得到每分钟的 **trade_volume, best_ask, size_best_ask, best_bid, size_best_bid**；  
  - 沿用现有「完整分钟网格 + 前向填充」，再算 mid_price、spread 等衍生特征；  
  - 构建**同一预测目标**：未来 30 分钟的 `mid_price`（价格预测）。  
- 输出：一份**分钟级表**（每行：date, stock_code, minute, 5 元特征 + 衍生特征 + 未来 30 分钟价格），并严格按时间划分：训练 = 1 月，测试 = 2 月第一周。

这样后续小模型、PatchTST、Informer、TimesFM 都共用同一份数据和同一套标签，保证可比性。

---

### 第二步：跑通小模型基线

- 用上一步的分钟级表：  
  - **LightGBM/XGBoost**：当前分钟 + 过去若干分钟的 5 元与衍生特征 → 预测 30 分钟收益或方向；  
  - **小 LSTM/TCN 或线性/AR**：滑动窗口（如 30–60 分钟）→ 同一预测目标。  
- 记录：**准确率 / MSE/MAE / R²**，以及**训练时间、推理延迟、内存占用**，便于后面和「大模型」比资源和效率。

---

### 第三步：接入 TimesFM（零样本/少样本）

- 安装：`pip install timesfm` 或 clone [timesfm](https://github.com/google-research/timesfm) 后按 README 安装 PyTorch 依赖。  
- 数据格式：  
  - 从分钟级表中，按 (date, stock_code) 取「过去一段」序列（如过去 512 或 1024 分钟，可先做等间隔采样以控制长度）；  
  - 每个变量一条序列（如 mid_price、或 5 元特征各一条），组成 `inputs=[...]`；  
  - 若模型只接受单变量，可先用 mid_price 或 30 分钟收益的滚动序列做点预测。  
- 调用：`model.forecast(horizon=30, inputs=[...])` 得到未来 30 步点预测（或更短 horizon 再聚合到 30 分钟收益）。  
- 把预测结果转成与你小模型**相同的标签**（例如 30 分钟收益或涨跌），在同一测试集上算**同一批指标**，并记录**推理耗时、GPU/内存**。

---

### 第四步：接入 PatchTST 或 Informer（二选一或都做）

- **PatchTST**：  
  - 把分钟级表整理成模型需要的 CSV 格式（多列 = 多变量，行 = 时间步）；  
  - 设置 look-back（如 60）、prediction length（30），在 1 月数据上训练，2 月第一周测试；  
  - 若用官方脚本，可先在一个小数据集（如单只股票）上跑通，再扩展到 15 只股票。  
- **Informer**：  
  - 同理，数据放进 `data/`，用 `--seq_len 60 --pred_len 30 --freq t` 等参数训练与预测；  
  - 输出与标签对齐到「未来 30 分钟收益/方向」，在同一测试集上评估。  
- 记录：**精度**（与第一步相同的指标）、**训练时间、推理延迟、显存/内存**，用于和 TimesFM、小模型对比。

---

### 第五步：对比与结论

- 做一张对比表：  
  - 行：LightGBM、小 LSTM、PatchTST/Informer、TimesFM、Chronos、Lag-Llama（及可选 DeepLOB）；  
  - 列：准确率（或 MSE/MAE）、训练时间、推理延迟、显存/内存（或成本）。  
- 若小模型在精度上**接近或优于** TimesFM / PatchTST / Informer，而**延迟和资源明显更小**，即可支撑你的论点：「在分钟级股票预测上，过度依赖大模型是对资源的浪费。」  
- 若某个大模型略优，可讨论**提升幅度是否值得**其额外计算与部署成本。

## 四、简要对照表

| 步骤 | 内容 | 产出 |
|------|------|------|
| 1 | 确认 5 元特征列、PySpark 分钟聚合与标签（未来 30 分钟收益/方向） | 分钟级表 + 训练/测试划分 |
| 2 | LightGBM + 小 LSTM/AR 等小模型 | 精度 + 时间/内存指标 |
| 3 | TimesFM 零样本/少样本预测 | 同一指标 + 推理资源 |
| 4 | PatchTST 或 Informer 在你数据上训练与预测 | 同一指标 + 训练/推理资源 |
| 5 | 对比表与结论 | 报告 / 论文图表 |

建议**先完成第一步**，再按 2 → 3 → 4 → 5 推进；TimesFM 与 PatchTST/Informer 可并行进行（不同机器或不同脚本即可）。
